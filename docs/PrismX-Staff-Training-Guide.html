What is PrismX?

PrismX is a real-time trade management system that receives precious metals trades from your WhatsApp, reviews them through a maker-checker pipeline, and dispatches them to your accounting software (OroSoft and SBS) via API.

Your daily workflow:

  • Staff sends a lock code via WhatsApp when a deal is closed
  • PrismX parses the code and creates a pending deal
  • A checker (Niyam or designated person) reviews and approves the deal
  • Approved deals flow to OroSoft (Pakka) or SBS (Kachha) for accounting

4 How to Review & Approve (Checker / Niyam)

  1. Open https://nt.areakpi.in/review
  2. The Pending tab shows new trades waiting for approval
  3. Each card shows:
    • Sender name and time
    • The raw WhatsApp message
    • Parsed fields: metal, quantity, rate, party, direction
    • Screenshot and OCR results (if an image was attached)
  4. Check the parsed fields — are metal, quantity, rate, and party correct?
  5. If something is wrong, toggle "Edit" and fix it before approving
  6. Click Approve for #NTK or #NTP deals
  7. For #NT (unclassified) deals, choose: "Approve as Kachha" or "Approve as Pakka"
  8. Click Reject if the trade is invalid
📝 The page auto-refreshes every 3 seconds. New trades appear within seconds of being sent. You don't need to manually refresh.
📝 Every approval and rejection is permanently logged in the Audit Trail. The system records who approved, when, and any changes made before approval.

6 Do's and Don'ts

✅ DO

  • Start every trade with #NTK, #NTP, or #NT
  • Use one space between each part of the code
  • Set opening stock on the very first day
  • Change your PIN from the default after first login
  • Check parsed fields before approving
  • Use the Edit toggle to fix errors before approving
  • Message +1 555 629 9466 for all trades

❌ DON'T

  • Don't send free-text messages to the bot — they'll be ignored
  • Don't use commas or special characters in the party name
  • Don't approve trades without checking the parsed fields
  • Don't share your personal PIN with others
  • Don't install WhatsApp on the bot number — it's API-only
  • Don't approve the same trade twice — the system prevents this

The /outbox page shows two panels side by side:

  • OroSoft Neo — Pakka deals ready to push via API
  • SBS — Kachha deals ready to push via API

Click "Send all" on either panel. The animated pipeline shows the progress. A Sync Log at the bottom records every API call with a sequential SYNC number (SYNC-0001, SYNC-0002, etc.) for reference with the software vendors.

📝 Concurrency protection: When you're dispatching, all other users see a pulsing banner: "Niyam is pushing 4 deals to OroSoft…" Their Send buttons are temporarily disabled to prevent conflicts.

9 Troubleshooting
